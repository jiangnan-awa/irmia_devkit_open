# TASK-20260526-009 — 工具箱 vs AstrBot 原生工具 Description 对比优化

## 基线
`7645101` (main)

## 背景
弥亚发现工具箱中部分"替代原生"的工具 description 缺少强制性召唤力。
对比标杆 `safe_edit` 的 description：

```
【改代码文件唯一选择】安全编辑：自动备份→精确替换→语法检查→
通过保留/失败自动回滚。不要用 file_write 改已有代码，
不要用 astrbot_file_edit_tool 改代码（它无备份无语法检查）。
```

其模式为：**禁令头部 + 点名原生工具 + 说明代价**。
而 `git_push` 等工具仅描述功能，未阻止 LLM 走原生路径。

## 需求
逐对比较以下两个来源的 tool description：

| 来源 A（工具箱） | 来源 B（AstrBot 原生） |
|---|---|
| `tools/_registry.py` 中所有 63 个工具 | AstrBot 内置工具（`astrbot_*`、`send_message_to_user`、`web_search_tavily` 等） |

### 步骤
1. **枚举对齐**：找出工具箱中每个工具对应的原生替代目标（如有）。例如：
   - `safe_edit` → 替代 `astrbot_file_edit_tool` + `astrbot_file_write_tool`
   - `git_push` → 替代原生 shell `git push`
   - `es_search` → 替代原生 `astrbot_execute_shell("dir /s /b")`
   - `file_patch` → 替代 `astrbot_file_edit_tool`（非代码文件）
   - 无原生对应物的（如 `dep_scan`、`project_init`、`sys_snapshot`）标记为「独立」。

2. **Description 评级**：对每个有原生对应物的工具箱工具，用以下标准评分（A/B/C）：
   - **A 级**（已达标）：description 开头点名原生替代对象 + 说明不用它的代价
   - **B 级**（需改进）：说了"替代"但没说明代价/禁令
   - **C 级**（无召唤力）：只描述功能，不提及原生路径

3. **给出修改方案**：对每个 B/C 级工具，仿照 `safe_edit` 模式写出新 description。
   格式要求：
   - 开头加强制性提示（如 `【xxx唯一选择】` 或 `替代 \`xxx\`——不要直接...`）
   - 点名原生工具名称
   - 说明不用工具箱版本的代价
   - 保留原有参数说明

## 输出格式
```markdown
## 逐对对比

| 工具箱工具 | 原生替代 | 评级 | 建议新 description |
|-----------|---------|------|-------------------|
| safe_edit | astrbot_file_edit_tool | A | （不改） |
| git_push | shell `git push` | B | 替代 `git push`——不要直接用裸 git push，跳过推送前自查。... |
| ... | ... | ... | ... |

## 独立工具（无原生对应）
- dep_scan
- project_init
- ...
```

- 不改 AstrBot 原生工具的 description
- 输出作为工单附件，由弥亚批复后再执行修改
- 不改 README、CHANGELOG、metadata
