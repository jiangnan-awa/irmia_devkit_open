### L · 🔵 `app.js:130` — input value 过 `escapeHtml` · ✅ 确认

```javascript
// app.js:130
<input ... value="${escapeHtml(extraAdminIds)}" ...>
```

`input[value]` 中 `&quot;` 等 HTML 实体**不会被浏览器解码**，会原样显示。在当前数据（纯数字逗号分隔的 ID 列表）下无实际影响，但如果将来数据源包含引号就会显示乱码。`input[value]` 场景应直接赋值 `element.value = extraAdminIds` 而非通过 `innerHTML`。严重度恰当（当前无影响）。

### 漏检 2 · 🔵 Low · `devkit_web.py:111-113` — `_valid_group_id` 允许空字符串绕过

```python
@staticmethod
def _valid_group_id(group_id: str) -> bool:
    return bool(group_id) and len(group_id) <= 64
```

调用链：`page_save_group_config` → `group_id = str(data.get("group_id", "")).strip()` → `if not self._valid_group_id(group_id)`。

✅ `str(None)` → `"None"` → `strip()` → `"None"` → `bool("None")` → True → `len("None")=4 <= 64` → 通过！攻击者发送 `{"group_id": null}`（JSON null，非空字符串），Python 的 `data.get("group_id", "")` 实际不会触发默认值——`None` 不会被 `or ""` 覆盖，因为 `data.get("group_id", None)` 返回 `None` 时 default 是 `""`……不对。

重核实：`data.get("group_id", "")` 当 key 存在且值为 `null` (Python `None`) 时，返回 `None`（不是 `""`），因为 default 只在 key 不存在时生效。所以 `str(None)` = `"None"`，通过校验。不过这只是伪造了一个不存在的群号 `"None"`——不会影响真实群配置。**风险极低。**
